# Product Philosophy Asset

## Asset Metadata

- `drop-id`: `drop-product-philosophy-core`
- `type`: `principle`
- `scope`: `well-global`
- `owner`: `product-core`

## First Principles

1. Assets are first-class citizens.
2. Explicit structure beats hidden context.
3. Every conclusion needs asset-backed evidence.
4. User intent stays authoritative, but may start incomplete.
5. One project serves one artifact.
6. Continuous re-evaluation is default behavior.
7. Conversation is a tool, not the default interface.
8. First-principles decomposition is required before execution commitments.
9. Asset boundaries should minimize blast radius during change.

## Product Implications

- every user input becomes a typed drop candidate
- AI-generated findings are also drops
- relations are visible and editable in UI
- gap claims must point to missing capability slots
- generation starts only after completeness threshold
- user should not need day-long prompt iteration loops
- unresolved intent assumptions must stay visible until confirmed
- repeated high blast radius should trigger asset partition refactor

## Non-Negotiables

- no keyword-list-only core capability
- no untraceable decision path
- no silent fallback that hides uncertainty
- no chat dependency for core project progression
- no blind execution from raw user wording without first-principles decomposition
- no persistent high-coupling asset graph without granularity refactor
